Block Descriptions

BlockPort(s)TechnologyResponsibility
Frontend 3000 3443 React 18 · Vite · Nginx Drag-and-drop MP4 upload · Prediction result with Grad-CAM overlay · Pipeline Dashboard (MLflow runs, Airflow DAG status, throughput). Nginx proxies /api/* to the backend and terminates TLS.
FastAPI Backend 8000 FastAPI · Uvicorn · Python 3.11 Core inference service. Receives MP4 uploads, runs MTCNN face detection + frame extraction, calls mlflow.pyfunc.load_model() in-process for inference, generates Grad-CAM heatmaps, detects feature drift, logs feedback, exposes Prometheus metrics at /metrics, and provides admin rollback at /admin/rollback.
MLflow Server 5000 MLflow v3.10.1 Experiment tracking UI + Model Registry. Stores run metadata, metrics, parameters, and artifacts. Manages model lifecycle: None → Staging → Production → Archived. Backend loads the Production model directly via mlflow.pyfunc.load_model("models:/deepfake/Production").
ML Training — PyTorch · EfficientNet-B0 · LSTM · MLproject CNN+LSTM architecture: EfficientNet-B0 extracts per-frame spatial features (1280-dim), 2-layer LSTM captures temporal dependencies across 30 frames, linear head outputs fake probability. Post-training INT8 dynamic quantization via ml/quantize.py. Every run tagged with Git commit SHA for reproducibility.
Apache Airflow 8080 Airflow 2.8.1 · LocalExecutor deepfake_pipeline (daily): Ingest MP4s → extract 30 frames → MTCNN face detection → EfficientNet feature extraction → schema validation → baseline stats → DVC versioning. retraining_dag (weekly): checks drift/staleness → triggers pipeline → runs MLflow training → evaluates F1 → registers if ≥ 0.90 → promotes to Production. LocalExecutor: no Redis/Celery worker needed.
Data Layer — DVC 3.x · Git LFS Manages versioning of large data artifacts: raw MP4s, extracted frames, face crops, and feature tensors. .gitattributes routes *.pt, *.mp4, *.jpg through Git LFS. dvc.lock pins exact data states for full pipeline reproducibility.
PostgreSQL 15 5432 PostgreSQL 15 Persistent store for Airflow metadata (DAG runs, task instances, XCom entries) and backend state. Health-checked via pg_isready before dependent services start.
Monitoring Stack 9090 3001 9100 Prometheus v2.50.1 · Grafana 10.3.1 · Node Exporter v1.7.0 Prometheus scrapes: backend /metrics (inference latency, error rate, drift score, prediction counts), Airflow /admin/metrics, Node Exporter (CPU/RAM/disk). Grafana provides 4 dashboards: Inference, Data Pipeline, Model Drift, Combined. Alert rules: error rate >5%, P95 latency >200ms, drift score >3.0, pipeline validation failure.
CI/CD Pipeline — GitHub Actions On push/PR to main: lint (black, flake8, isort, ESLint) → unit tests with coverage → dvc repro --dry (DAG integrity) → integration tests → Docker build. Ensures every merge is tested and the DVC pipeline is consistent.
