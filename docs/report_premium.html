1. Executive Summary

The Deepfake Detection System is a production-grade, end-to-end MLOps platform that classifies MP4 videos as real or AI-generated (fake) using a deep learning model. The system is designed for fully local, on-premise deployment using Docker Compose — no cloud dependency exists anywhere in the stack.

The platform covers the full ML product lifecycle: automated data ingestion and preprocessing, model training with full experiment tracking, model registry with staged promotion, real-time inference via a REST API, a React frontend with explainability (Grad-CAM), comprehensive Prometheus/Grafana monitoring with drift detection, automated weekly retraining, and a CI/CD pipeline enforcing code quality and reproducibility on every commit.

2. Problem Statement

2.1 Business Context

Deepfake videos — AI-synthesised media where a person's face is replaced with another — pose serious threats to trust, journalism, legal evidence, and personal privacy. Detection tools have traditionally required deep ML expertise to operate. This project builds a production-ready tool that any non-technical user can operate through a web browser.

Design Decision — Container Optimisation
Three containers were eliminated: mlflow-serve (9.36 GB, never called), redis (61 MB, Celery broker only), and airflow-worker (switched to LocalExecutor). No functionality was lost; total image footprint was significantly reduced.

4. ML Model Design

4.1 Architecture

The model uses a CNN + LSTM architecture to detect deepfakes by analysing both spatial (per-frame) and temporal (cross-frame) features:

Input: (batch, 30, 3, 224, 224) — 30 face-cropped, normalised frames │ ▼ EfficientNet-B0 (pretrained ImageNet, frozen) — spatial feature extraction per frame — output: (batch, 30, 1280) │ ▼ 2-layer LSTM (hidden=256, dropout=0.3) — temporal dependencies across 30 frames — captures manipulation artefacts over time — output: (batch, 256) ← last hidden state │ ▼ Linear(256→128) → ReLU → Dropout(0.3) → Linear(128→1) → Sigmoid │ ▼ Output: (batch, 1) probability ≥ 0.5 → FAKE probability < 0.5 → REAL

4.2 Design Rationale

ChoiceDecisionRationale
Feature extractorEfficientNet-B0Best accuracy/compute ratio; 1280-dim features per frame; pretrained ImageNet weights
Temporal model2-layer LSTMCaptures inter-frame temporal artefacts; simpler than transformers for 30-frame sequences
CNN frozenYes106 training samples — CNN fine-tuning causes score collapse; frozen + LSTM is stable
Face detectionMTCNNHigh accuracy across poses; fallback to resized full frame when no face detected
Threshold0.5 + best F1 sweepfind_best_threshold() sweeps 0.1–0.9 to find optimal F1 threshold per evaluation run

4.4 Model Optimisation — INT8 Quantisation

For CPU-only deployment, the model is quantised to INT8 using PyTorch dynamic quantisation. This reduces LSTM and Linear layer storage without requiring a calibration dataset:

# ml/quantize.py
quantized = torch.quantization.quantize_dynamic(
    model,
    {torch.nn.Linear, torch.nn.LSTM},
    dtype=torch.qint8,
)

4.5 Preprocessing Pipeline

MP4 file │ ├─ extract_frames(): OpenCV samples 30 frames evenly via np.linspace │ raises ValueError if video unreadable or has 0 frames │ ├─ detect_faces(): MTCNN face detection on each frame │ fallback: resize full frame to 224×224 if no face found │ └─ normalize(): ImageNet mean/std → torch.Tensor (30, 3, 224, 224) mean = [0.485, 0.456, 0.406] std = [0.229, 0.224, 0.225]

4.6 Explainability — Grad-CAM

Every prediction generates a Grad-CAM heatmap showing which regions of the face influenced the model's decision:

  • Hooks the last EfficientNet _blocks layer
  • Computes gradients of output w.r.t. activation maps
  • Applies ReLU + normalisation → 224×224 heatmap
  • Overlays on middle frame → returns base64 PNG
  • Returns "" on any failure (best-effort, never blocks inference)

7.2 POST /predict — Data Flow

1. Validate: file.filename.endswith(".mp4") → 400 on failure 2. Save to tempfile.NamedTemporaryFile(suffix=".mp4") 3. preprocess_video(tmp_path): extract_frames() → detect_faces() → normalize() → Tensor(30,3,224,224) 4. model.predict(tensor.unsqueeze(0).numpy()) → confidence float 5. prediction = "fake" if confidence ≥ 0.5 else "real" 6. compute_drift_score(frame_mean_feature, baseline) → z-score → if z-score > 3.0: DRIFT_DETECTED.inc(); logger.warning(...) 7. generate_gradcam(pytorch_model, frames_tensor) → base64 PNG 8. Update all 22 Prometheus metrics 9. Return PredictResponse JSON

9.3 Drift Detection

1. Compute mean feature vector across 30 frames: shape (3, 224, 224) 2. Load baseline from ml/feature_baseline.json (generated by Airflow) 3. z-score = mean(|features - baseline_mean| / max(baseline_std, 1e-8)) 4. If z-score > 3.0 → increment DRIFT_DETECTED counter → emit structured warning log → update DRIFT_SCORE Prometheus gauge → fire Grafana alert 5. Weekly: retraining_dag checks baseline age If staleness > 7 days → trigger full retraining pipeline

Key Insight — MLflow APIification
The project intentionally uses mlflow.pyfunc.load_model() in-process rather than a separate mlflow models serve container. This is superior: lower latency (no HTTP hop), fewer failure points, and the MLflow Model Registry still fully manages versioning and promotion. FastAPI IS the model server.

14. Known Limitations & Future Work

14.1 Current Limitations

AreaLimitation
Training data106 videos — model generalises well but would benefit from a larger, more diverse dataset
GPU supportDesigned for CPU inference; training supports CUDA but not tested under load
Batch inference/predict/batch limited to 10 files; no async queue
E2E tests in CIRequire live Docker stack — skipped in CI; run manually
Face detection fallbackMTCNN no-face fallback to full frame may reduce accuracy for distant shots

14.2 Planned Improvements

ItemDescription
Background inference queueReplace synchronous /predict with async task queue (Celery/Huey) for large files
Progressive fine-tuningUnfreeze EfficientNet-B0 last 3 blocks after LSTM warms up
Video TransformerReplace LSTM with a lightweight Video Transformer for better temporal modelling
Dataset expansionIntegrate FaceForensics++ and Celeb-DF for broader manipulation coverage
Canary deploymentA/B testing between model versions before full Production promotion
